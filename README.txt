Un problème recensé est la durée d'exécution assez longue des algorithmes de planification 
quand on a nombre de pièce supérieur ou égale à 6.

Nous avons rajouté la classe BacktrackAllSolver qui permet de retourner toutes les solutions 
possibles d'un modèle.

Nous avons aussi décider d'avoir 10 instances dans notre de base de données afin d'avoir
plus de règles renvoyé par notre extracteur plutôt que de retourner toutes les solutions
possibles d'un modèle, ce qui augmente grandement le temps d'exécution de la recherche.
